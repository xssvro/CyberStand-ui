# CyberStand UI 设计理念

面向 AI 协作与可主题化的**通用 UI 体系**（设计 token 与组件理念可映射到多端实现）。本文档约定**视觉与交互层面的设计方向**；实现以 `src/core/vars.css` 中的 **CSS Variables** 为唯一真源，文档与组件样式应与之对齐。

当前版本**优先固化色彩思路**；版式、字号、动效等将在后续迭代中补充。

---

## 艺术取向与灵感来源

### 为何借用赛博朋克

赛博朋克在视觉上长期处理同一组矛盾：**高科技与低生活**、**冷系统与体温**、**可读信息与过量噪声**。界面设计若借用这一传统，并不是为了贴标签，而是借用一套已被反复验证的**光色语法**：深色环境压暗视觉噪音，霓虹与高光只打在「可操作」与「可阅读」之处——这与组件库「层次清晰、状态可辨」的目标同构。

以下作品并非逐帧复刻，而是作为**调色与氛围的参照系**；具体实现仍服从可读性、无障碍与 token 约束。

### 经典作品的视觉关键词

**《银翼杀手》（1982）**  
雨雾中的洛杉矶、霓虹被湿气晕开、室内偶有钨丝暖光。启示在于：**环境极暗，光源极少且语义明确**——对应我们的多层深色底与「只有少数色相被点亮」的按钮/焦点策略。

**《银翼杀手 2049》**  
琥珀与橙雾、废土尺度、建筑压迫感下的零星色块。启示在于：**低饱和环境里，一点饱和色（尤其橙琥珀系）即成为叙事锚点**——对应主色黄、信息蓝在界面中只做「锚点」，而非铺满。

**《阿基拉》（AKIRA，1988）**  
新东京的夜色、车流尾灯与警示红、力量濒临失控前的屏息与爆发。启示在于：**高饱和红只留给「危险 / 临界」式的关键状态**，绝大部分界面仍压在冷色底上——对应错误与珊瑚系的克制度，主操作与信息锚点仍以黄、蓝为主，避免整屏泛红。

**《攻壳机动队》**（押井版等）  
青绿 HUD、透明叠层、城市信息密度。启示在于：**冷色描边与界面灰偏同一色相**、信息层可叠——对应 slate 正文、半透明边框与后续可扩展的层级 UI。

**《铳梦》**  
废铁、肉与机械并置，粉与冷金属的反差。启示在于：**粉色不必是「甜腻」，可以是梦、机体与危险隐喻**——对应珊瑚系次要/错误与冷底的并置，而非滥用大红。

**《赛博朋克 2077》**  
夜之城黄、洋红与青蓝的霓虹三角、UI 即世界观。启示在于：**品牌黄与信息蓝可并存**，暖点缀与冷界面分工明确——对应 Primary / Info / Secondary 的分工（实现只用 token 实色，不用渐变铺底）。

### 从灵感到 token 的对应（摘要）

| 感受来源 | 在 CyberStand UI 中的落点 |
|----------|----------------------|
| 雨夜、雾、被压暗的城市 | 多阶深色背景、避免整块纯黑扁平 |
| 霓虹只点关键处 | 主色黄、信息蓝、珊瑚仅用于语义与强调 |
| HUD、全息、界面叠在世界上 | 冷灰正文、半透明边界、焦点辉光 |
| 废土与机体 | 珊瑚/错误与冷底对比，不作为大面积铺色 |
| 暴走与警示红（阿基拉） | 高饱和暖色仅用于错误/次要强调，不作环境主色 |
| 夜之城式品牌记忆 | 黄与蓝可同框，但正文仍以 slate 为主 |

### 其他可参考的作品与传统

下列作品同样常被纳入「赛博朋克」或**强信息社会 / 高对比城市夜景**的讨论，可作氛围延伸；与上文**主要灵感条目**并列阅读即可，不必全部映射到 token。

| 作品 | 可借用的视觉或叙事线索 |
|------|-------------------------|
| **《神经漫游者》（小说，威廉·吉布森）** | 铬、矩阵、数据的文学源头——**界面即隐喻**：颜色应服务语义，而非装饰。 |
| **《黑客帝国》三部曲** | 数字雨、终端绿、红蓝药丸的**二元抉择**——适合联想「默认 / 危险 / 信息」等清晰分叉，而非照搬绿色 UI。 |
| **《玲音》（serial experiments lain）** | 极简界面、白与空、网络人格分裂——可作**留白与层级**的反面参照：我们仍保持深色底上的可读密度。 |
| **《大都会》（1927）** | 垂直城市、灯光分层——早期「巨构 + 霓虹分层」原型，与**多阶背景**思路相通。 |
| **《心理测量者》** | 冷蓝执法 UI、系统伦理——强化「冷灰 + 蓝系信息」的**可信赖感**与规则感（非照搬色相）。 |
| **《副本》（Altered Carbon）** | 霓虹雨、赛博旅馆、肉体可换——**环境暗、点缀亮**的夜城语法，与银翼系一脉。 |
| **《特警判官》（2012）** | 高对比、窄巷与巨塔——**单块高饱和 + 大面积压暗**，与「锚点色」策略一致。 |
| **《赛博朋克：边缘行者》** | 与 2077 共享世界观，更强调**个体在系统下的荧光**——适合理解「少数亮色即情绪锚点」。 |

文学与游戏设定集（如《神经漫游者》之后的赛博朋克文学谱系、TRPG《赛博朋克》系列）则补充**语汇与伦理维度**：界面是工具，也是世界观的一小块碎片。

## 色彩

### 设计取向

整体气质偏向 **赛博朋克 / 夜之城**：深色冷底、霓虹点缀、信息结构清晰；其艺术脉络见上一节，色彩需同时满足：

- **可读性**：长文案与组件状态在深色背景上对比充足。
- **一致性**：界面灰与底板同色相（偏冷、略偏蓝），避免暖灰与冷底「打架」。
- **可扩展**：语义色与中性色均提供阶梯，便于 hover、禁用、柔和底等状态。

### 深色背景层级

背景不是单一黑，而是 **5～6 个亮度台阶**，从环境到浮起表面逐级变亮，层次柔和、边界多用低透明度冷色描边而非死灰实线。

| Token | 说明 | 参考 |
|--------|------|------|
| `--su-bg-page` | 最外环境色 | `#05060b` |
| `--su-bg-default` | 主阅读区 / 内容槽 | `#0b0e16` |
| `--su-bg-subtle` | 略浅一层分区 | `#10141e` |
| `--su-bg-muted` | 输入、斑马纹等 | `#161b27` |
| `--su-bg-elevated` | 侧栏、卡片等「浮起」面 | `#1e2533` |
| `--su-bg-code` | 代码块最深底 | `#080a12` |

### 品牌与语义色

- **主色 Primary（黄）**：`#fcee0a` 一带，用于主按钮、高识别操作；实心按钮上文字用 `--su-on-primary`（近黑），保证对比。
- **次要 Secondary（珊瑚）**：`#e66054`，偏粉一点的珊瑚，与主色黄、信息蓝形成三角点缀，用于次要强调、部分错误态。
- **信息 Info（蓝）**：`#21aadd`，界面中「信息、链接、焦点」的冷色锚点；焦点环、输入聚焦等与该色一致。
- **成功 / 警告 / 错误**：绿色、橙色、与珊瑚系错误色，保持语义可辨；错误与 secondary 同系时，通过上下文与文案区分。

品牌联想可用黄 / 蓝 / 珊瑚 **分段实色**（如并列色块、插画），**不**以渐变作为产品界面与文档壳层的默认手段；正文与组件仍只服从 token。

### 文案色（通用阅读）

正文与界面灰 **统一为偏蓝的 slate**，与冷色底板、半透明边框（`rgba(200, 220, 255, …)`）同一气质，**不再使用暖米灰系**作为默认正文。

| Token | 用途 | 深色默认 |
|--------|------|-----------|
| `--su-text-default` | 主文案、标题与正文 | `#cdd5e3` |
| `--su-text-muted` | 次要说明、辅助导航 | `#94a0b3` |
| `--su-text-subtle` | 更弱层级、占位提示 | `#6b7585` |

### 重点与强调

- **`--su-text-emphasis`**：`#f9d15c`，用于「重点说明」类文案（如 Markdown 加粗、提示标题），与主色黄呼应但不替代主色 token 的全局用法。
- 浅色主题下可为略深的琥珀色，保证在浅底上的对比度。

### 边框与焦点

- 边框以 **低透明度冷色** 分层（`subtle` / `default` / `strong`），避免粗硬灰线。
- 焦点与键盘导航：**信息蓝**发光（`--su-focus-ring-color` 等），与 `--su-info-500` 一致。

### 浅色主题

通过 `[data-theme='light']` 覆盖背景与文案：浅灰底、冷灰字（如 `#2a303c` / `#5a6270`）。**主色与深色同为黄色锚点**（浅色下 `--su-primary-600` 约 `#ebdb00`，接近夜城黄、略压一档以利小字对比）；语义色阶梯仍用同一套变量名。组件颜色一律来自全局 `--su-*`，勿在业务里写死主色 hex。

### 技术约定

- 业务与组件样式 **只引用** `--su-*` 变量，避免硬编码十六进制。
- **界面与文档站壳层**以 **实色表面 + token 边框** 为主，**不使用背景渐变**作环境底或装饰条；营销页若需渐变须单独约定，不进入组件与文档预览。
- 新增主题或改色时，**先改 `vars.css`**，再同步本文档与示例。
